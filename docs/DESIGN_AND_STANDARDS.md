# Design And Coding Standards

## Design Focus
- Keep UI abstractions generic and composable.
- Prefer data-driven rendering surfaces over rigid field layouts.
- Keep control-plane behavior simple and deterministic.

## Coding Standards
- Keep modules small and single-purpose.
- Avoid hidden coupling between UI widgets and data contracts.
- Update docs when behavior changes.
- Do not revert unrelated user changes.
- Prefer small, reviewable commits.
